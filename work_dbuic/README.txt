このZIPの中身
--------------

js/plugins/DB_UIComposer.js
  RPGツクールMZ側に入れる再現プラグインです。

DB_UIComposer_Tool/index.html
  ブラウザで開く配置編集ツールです。

data/uiLayouts/SampleMapUI.json
  LoadLayoutFile 用のサンプルJSONです。

data/uiLayouts/DebugLayerTest.json
  ウィンドウ外描画、表示順、装飾画像の再現確認用JSONです。

導入方法
--------

1. js/plugins/DB_UIComposer.js を、RPGツクールMZプロジェクトの js/plugins/ にコピーします。
2. ツクールのプラグイン管理で DB_UIComposer をONにします。
3. DB_UIComposer_Tool/index.html をブラウザで開きます。
4. 必要に応じて「ツクールプロジェクト読込」でプロジェクトフォルダを指定します。
5. ツールでUIを配置します。
6. 「MZスクリプトをコピー」を押し、イベントコマンド「スクリプト」に貼り付けて実行します。

基本方針
--------

・配置編集は外部ツール側で行います。
・MZ側プラグインはJSONを読み込んで再現します。
・画像はツクールプロジェクト読込後、「プロジェクト画像から選択」で指定します。
・データ保存/データ読込では、対応ブラウザの場合、プロジェクトフォルダの再読込も試みます。
・ゲーム画面配置モードでは、重なったウィンドウをダブルクリックすると手前から奥へ選択を巡回します。
・ウィンドウ内配置モードでは、重なったパーツ/ウィンドウをダブルクリックすると、現在フォーカス中の対象を基準に次の奥側候補へ巡回します。
・ウィンドウ内配置モードでは、編集対象のウィンドウと内部パーツをツールプレビュー上だけ一時的に最前面へ出します。実際の表示順・レイヤー値は変更されません。

デバッグ手順
------------

1. MZ側でプラグインコマンド「デバッグログ切替」を実行し、ONにします。
2. 通常どおり ApplyLayoutJson または LoadLayoutFile でUIを表示します。
3. F8/F12などで開発者ツールのConsoleを開きます。
4. 必要に応じて「デバッグ情報出力」コマンドを実行します。

表示順確認用テストは、以下を LoadLayoutFile で読み込んでください。

DebugLayerTest.json

主なプラグインコマンド
----------------------

ApplyLayoutJson
  JSON文字列からUIを一括作成します。

LoadLayoutFile
  data/uiLayouts/ 内のJSONファイルを読み込みます。

SetDebugLog
  MZ実機側のデバッグログ出力をON/OFFします。

DumpDebugInfo
  現在保持しているレイアウト、ウィンドウ、パーツ情報をコンソールへ出力します。

ClearAll
  作成済みUIをすべて削除します。

ShowLayout / HideLayout
  レイアウト単位で表示/非表示を切り替えます。

SetWindowVisible
  指定ウィンドウだけ表示/非表示を切り替えます。

CreateWindow / RemoveWindow
  単体ウィンドウを作成/削除します。

AddText / AddImage / AddGauge / AddButton
  指定ウィンドウ内にパーツを追加します。

RemoveItem / ClearWindowItems
  指定パーツの削除、またはウィンドウ内パーツの一括削除を行います。

注意
----

このツールはブラウザのFile System Access APIを利用します。
ブラウザや権限状態によっては、プロジェクトフォルダの自動再読込ができない場合があります。
その場合は再度「ツクールプロジェクト読込」でフォルダを指定してください。

・画像候補は、プロジェクト読込済みならimg配下の画像を候補に含めます。未読込の場合は現在のレイアウトで使用している画像のみ候補になります。
